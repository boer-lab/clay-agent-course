# Two things to take with you

Both of these came out of building this course. Neither is a summary of it. They are the things I
wish I had on the first day, and they are yours under CC BY.

## The cheatsheet — hand this to your agent, not to yourself

This one is written for Claude Code to read, not you. It is every command that exists, the shape
claygent's output comes back in, the nineteen ready-made enrichments you can call by name, and the
handful of mistakes that cost me hours. The wrong filter field that returns an empty list instead of
an error. The command that needs two ids and complains about the wrong one. The research question
that lives in a shared setting rather than inside your workflow.

**How to use it:** download it, then either paste the whole thing into Claude Code at the start of a
session, or save it in your project folder as `CLAUDE.md`, which is the file Claude Code reads
automatically when it opens. Then when you ask for something in Clay, your agent already knows the
traps instead of finding them the expensive way. If it ever tells you something that contradicts the
cheatsheet, believe the cheatsheet and check the date at the bottom of it.

That is the whole point. Most of the time you lose with an agent is not spent building. It is spent
watching it discover a limit you already knew about.

## The starter prompts — for you

Every sentence this course had you say out loud, on one page, with blanks where your own details go.
Connect, find companies, build the workflow, run it, check its work, change the question, and the
two lines that get you out of trouble when something looks wrong.

**How to use it:** keep it open in a tab the first few times. Copy a line, fill in the blanks, send
it. After a week you will not need it, which is the correct fate for a page like this.
